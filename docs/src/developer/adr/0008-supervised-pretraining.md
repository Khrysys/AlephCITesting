# 0008 - Supervised Pretraining on Lichess Evaluated Games

## Context

Self-play training from random network initialization is viable but slow. A randomly initialized network produces near-random move probabilities and value estimates, which means early self-play games are effectively random walks through legal positions. The training signal from these games is weak — the network has no basis for evaluating positions or preferring good moves, so the positions generated during search are low-quality and the resulting gradient updates are noisy. Progress out of the random-play regime is gradual and compute-intensive.

Supervised pretraining on a large database of evaluated positions bypasses this cold start entirely. The network is initialized to a reasonable policy and value function before self-play begins, which means the first self-play games are played at a meaningful level and generate a useful training signal immediately. Pretraining on a static dataset is also significantly faster than self-play — there is no search overhead, no game generation loop, and no SPRT evaluation between training steps.

## Decision

Aleph pretrain on the Lichess evaluated game database, which contains approximately 150 million positions with Stockfish evaluations. The database is preprocessed before training; the raw data is not used directly.

### Preprocessing

**Value head target:** Stockfish centipawn evaluations are converted to WDL probabilities. The conversion maps centipawn scores to a win/draw/loss distribution using a sigmoid-based formula, producing soft targets rather than hard win/loss labels. This gives the value head a smooth training signal that reflects evaluation uncertainty near equality.

**Policy head target:** Stockfish is run locally on each position in MultiPV mode to produce a full move score distribution over legal moves. A one-hot policy (top move only) is not used — it would reduce the policy target to pure imitation learning with no uncertainty over alternatives, which would produce an overconfident policy head that assigns near-zero probability to all non-top moves regardless of their objective quality.

**Moves-left head target:** Game lengths are pulled from the Lichess database PGN data. The moves-left target for each position is the number of moves remaining in the game from that position. This is an approximation — the actual game length depends on the players involved and is not the same as the engine-optimal game length — but provides a reasonable initialization for the moves-left head. The exact training approach for moves-left during pretraining is an open item.

### Transition to self-play

Once pretraining is complete, training transitions to self-play with search-contempt as described in ADR-0002. The pretrained network serves as the starting point for the first generation of self-play games.

## Consequences

**Positive**

- Avoids the random-play cold start. The network enters self-play with a meaningful policy and value function, producing useful training signal from the first generation of self-play games.
- Pretraining on a static dataset is significantly faster than self-play — no search overhead, no game generation loop, and no SPRT evaluation between steps.
- The Lichess database is large (~150M positions) and diverse, covering a wide range of openings, middlegames, and endgames across many skill levels.
- Soft WDL targets from centipawn conversion provide a smoother value training signal than binary game outcome labels.
- MultiPV policy targets give the policy head a calibrated distribution over legal moves rather than a one-hot approximation.

**Negative**

- Stockfish's policy distribution reflects Stockfish's evaluation function, not Aleph's. The pretrained policy may encode biases specific to Stockfish's search and evaluation that are not optimal for a neural network evaluator.
- Centipawn to WDL conversion is an approximation. The conversion formula introduces its own assumptions about the relationship between evaluation and game outcome that may not hold uniformly across all position types.
- Running Stockfish in MultiPV mode on ~150M positions is a significant preprocessing cost. This is a one-time cost but must be accounted for in the overall compute budget.
- Game lengths from the Lichess database reflect human play, not engine-optimal play. The moves-left head will be initialized to human game length distributions, which may differ substantially from the lengths of self-play games generated by Aleph.
- The transition from supervised pretraining to self-play is a discrete switch. There is no curriculum or mixing schedule between the two phases; any distribution shift at the transition point is absorbed by the network during early self-play.
